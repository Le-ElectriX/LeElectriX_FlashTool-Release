# 更新日志

本文档记录 LeElectriX FlashTool 各版本中面向用户、集成方和贡献者的重要变化。格式参考 Keep a Changelog，并按版本倒序排列。

## v1.2.6.0 - 2026-07-13

### 修复

- 修复 Enpower Excel 参数导入遇到空白行、缺失单元格或混合单元格类型时解析失败或提前结束的问题。
- 修复进入 MCU/MCU 扩展标定页后，页面轮询可能自动打开 CAN 硬件通道的问题；页面现在只使用已经打开的通道。
- 修复 Enpower 标定将空响应或短响应误判为握手失败，以及未按完整 12-bit 参数号核对写响应的问题，避免无效重握手和串线响应被误判为写入成功。
- 修复 TXT 批量下载期间轮询、初始化与写入交错造成的参数漏发、乱序或失败未上报问题；批量写入现在按完整事务执行，单项连续失败会终止后续下载，异常和页面停止也不会遗留占用或继续发送。

### 变更

- 移除 MCU 参数页面列表底部多余的空白行。

## v1.2.5.0 - 2026-07-02

### 新增

- 新增基于严格四段式 Git 标签（`vX.Y.Z.W`）的自动版本计算，统一应用显示、程序集版本、安装包版本及发布包命名。

### 修复

- 修复 Enpower 标定页面离开后旧实例仍继续轮询，以及普通参数页和默认参数页共享状态造成相互干扰的问题；页面现在拥有独立生命周期，并能正确恢复发送/接收 CAN ID 和握手状态。
- 修复导航失败、后退或回滚后原页面生命周期和导航参数恢复错误的问题。
- 修复更新包文件名或发布名称异常时的版本解析与比较问题，并避免网络、共享目录或更新检查异常阻塞应用启动。

### 安全

- 移除更新检查代码中硬编码的 GitHub 访问令牌，改用无需私有凭据的公开发布信息接口。

## v1.2.4.0 - 2026-05-29

### 修复

- 修复 Enpower 默认参数页中 Application 版本为 `0` 时显示失真的问题；此时改为显示对应的 Library 版本。

## v1.2.3.0 - 2026-05-14

### 新增

- 为 MCU 默认参数页中的关键阈值编辑增加高级功能解锁保护，降低误改关键参数的风险。

### 修复

- 修复 Enpower 刷写流程在真正发送 `RequestDownload` 请求前就检查旧执行结果的问题。
- 扩展 EPS650 固件识别与刷写支持，增加 ChipCode `0x71` 及起始地址 `0x3EA000`、容量 `0xC000` 的 HEX 布局，并补充相应 CAN 请求/响应 ID 映射。

### 变更

- 将解决方案、项目、测试、驱动和文档等主体内容统一迁移到 `src/` 目录，调整仓库构建入口。

## v1.1.2.2 - 2026-04-15

### 新增

- 重组解决方案结构，补充独立 Core 项目、MSTest 测试项目、项目 README 和使用说明资源，并调整便携版发布目录。

### 修复

- 修复扫描 ZLG 设备时可能将可打开但类型不匹配的设备错误加入列表，导致后续按错误设备类型初始化并无法打开 CAN2 通道的问题。

## v1.1.2.1 - 2026-04-13

### 修复

- 恢复 Enpower 扩展会话的较长重试窗口，为强制刷写时手动重新上电预留足够时间，避免过早判定无响应并终止刷写。

## v1.1.2.0 - 2026-04-09

### 变更

- 完善应用、CAN-TP 和 Enpower 刷写日志，统一通过依赖注入的日志接口记录来源上下文，使通信与刷写问题更易定位。

## v1.1.1.0 - 2026-04-09

### 新增

- 增加 Enpower 刷写状态、关键操作和分步骤失败原因日志，并在设置页提供日志目录入口。
- 默认禁用刷写 ID 扫描，增加一次性密码控制的高级功能解锁入口，降低非授权扫描风险。

### 修复

- 将 Enpower 擦除例程等待时间统一延长到 4 秒，降低控制器擦除耗时较长时的响应超时概率。
- 缩短普通无通信场景下扩展会话的失败判定时间，并为会话切换、解锁、擦除、传输、校验和复位等阶段提供更明确的错误提示。

### 变更

- 将日志保存位置迁移到用户本地应用数据目录，继续按 App、CAN 和 DoCAN 分类滚动保存。

## v1.0.9.1 - 2026-04-08

### 修复

- 修复 ZLG 批量接收报文时所有结果错误复用第一帧 CAN ID 的问题，确保每帧保留自己的实际 ID。

### 变更

- 将应用目标框架升级到 .NET 8，并启用可空引用类型等编译配置。

## v1.0.9.0 - 2026-01-29

### 新增

- 引入 Serilog 日志系统，将应用日志、原始 CAN 报文和 DoCAN 报文分别滚动写入 App、CAN 和 DoCAN 日志文件。

### 修复

- 修复 CAN 接收缓冲区清理标志在线程间可能不可见、导致清理流程陷入死循环的问题。

## v1.0.8.0 - 2025-12-15

### 新增

- 增加 EPS650 联合开发产品刷写支持，可识别并处理起始地址 `0x3E8000`、容量 `0xE000` 的固件布局，并适配其版本信息、刷写解锁和例程等待要求。
